10 Step program to understanding CSS position

This tutorial examines the different layout properties available in CSS: position:static, position:relative, position:absolute, and float.

1. position:static

The default positioning for all elements is position:static, which means the element is not positioned and occurs where it normally would in the document.

Normally you wouldn't specify this unless you needed to override a positioning that had been previously set.

#div-1 {
position: static;
}

Static positioned elements are not affected by the top, bottom, left, and right properties.

The layout on the right, contains 3 outer divs

  • div-before - purple
  • div-1 - black
  • div-after - purple
and div-1 contains 3 inner divs
  • div-1a - red
  • div-1b - green
  • div-1c - blue
The remainder of the demonstration will operate on these divs to demonstrate the behavior

2. position:relative

If you specify position:relative, then you can use top or bottom, and left or right to move the element relative to where it would normally occur in the document.

Let's move div-1 down 20 pixels, and to the left 40 pixels:

#div-1 {
position: relative;
top: 20px;
left: -40px;
}

Notice the space where div-1 normally would have been if we had not moved it: now it is an empty space. The next element div-after did not move when we moved div-1. That's because div-1 still occupies that original physical space in the document, even though we have moved it visually.

It would appear that position:relative is not very useful, but it will perform an important task later in this tutorial.

3. position:absolute

When you specify position:absolute, the element is removed from the document and placed exactly where you tell it to go (provided the outer div is not being positioned as relative).

The inner divs div-1b and div-1c are still positioned static as the default - and only div-1a has been positioned absolute.

Let's move div-1a to the top right of the page:

#div-1a {
position: absolute;
top: 0;
right: 0;
width: 200px;
}

Notice that this time, since div-1a was removed from the document, the other elements on the page were positioned differently: div-1b, div-1c, and div-after moved up since div-1a was no longer there.

Also notice that div-1a was positioned in the top right corner of the page. It's nice to be able to position things directly the page, but it's of limited value.

What we really want is to position div-1a relative to div-1. And that's where relative position comes back into play.

Footnotes

  • There is a bug in the Windows IE browser: if you specify a relative width (like "width:50%") then the width will be based on the parent element instead of on the positioning element.

4. position:relative + position:absolute

If we set relative positioning on div-1, any elements within div-1 will be positioned relative to div-1. Then if we set absolute positioning on div-1a, we can move it to the top right of div-1:

#div-1 {
position: relative;
}
#div-1a {
position: absolute;
top: 0;
right: 0;
width: 200px;
}

5. two column absolute

Now we know th effect of absolute, why not simply make a multi-column layout within the containing div.

If we apply absolute positioning to the inner divs and relative the outer div, we should have columns!

#div-1 {
position: relative;
}
#div-1a {
position: absolute;
top: 0;
right: 0;
width: 200px;
}
#div-1b {
position: absolute;
top: 0;
left: 0;
width: 200px;
}

One advantage to using absolute positioning is that we can position the elements in any order on the page, regardless of the order they appear in the HTML. So I put div-1b before div-1a.

Houston we have a problem! What happened to the other elements?

They are being obscured by the absolutely positioned elements. What can we do about that?

6. two column absolute height

One solution is to set a fixed height on the elements.

But that is not a viable solution for most designs, because we usually do not know how much text will be in the elements, or the exact font sizes that will be used.

#div-1 {
position: relative;
height: 250px;
}
#div-1a {
position: absolute;
top: 0px;
right: 0px;
width: 200px;
}
#div-1b {
position: absolute;
top: 0px;
left: 0px;
width: 200px;
}

7. float

For variable height columns, absolute positioning removes the nodes from the normal document flow and all remaining items flow up naturally, so for some developers this may appear as if absolute position does not work exactly as planned, so let's come up with another solution.

We can apply a "float" position on an element to push it as far as possible to the right or to the left, and allow text to wrap around it. This is commonly seen for images, however, for our demonstration but we will use it for more complex layout tasks (because it's the only tool we have).

#div-1a {
float: left;
width: 200px;
}

In this exampe we had applied a position of float to the div-1a element, pushing the div to left within the parent div-1 container element. The remaining elements are statically placed (default position) and wrap around the floated element.

8. float columns

If we float one column to the left, then also float the second column to the left, they will push up against each other.

#div-1a {
float: left;
width: 150px;
}
#div-1b {
float: left;
width: 150px;
}

The effect we see is all the remaining divs are still statically positioned and move upward. For example if we examine div-1c it is placed to the right of second floated element. And the div-after is moved up - essentially closing off the height of the 3 inner divs.

The 2 floated elements div-1a and div-1b extend their respective heights without interference from the remaining divs.

The question may come up that we are now in the same position we were when absolute position.

Well not exactly, in this case we can see the div-1c and the div-after and are not be obscurred.

9. float columns with clear

So how do we position the inner element div-1c below the floated elements and not adjust the height as being fixed.

The technique would simply be after the floating elements we can "clear" the floats to push down the rest of the content.

#div-1a {
float: left;
width: 190px;
}
#div-1b {
float: left;
width: 190px;
}
#div-1c {
clear: both;
}

What we observe from this is inner element div-1c clears the floats and will placed just under the floating elements

Notice: We do not apply clear:both; to the div-after element.
